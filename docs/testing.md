# Testing

HERMES includes a characterization test suite that protects current scientific behavior while the code is cleaned and unified.

## Run Tests

Activate the environment and run pytest.

```bash
conda activate hermes
python -m pytest
```

The expected result for this draft is:

```text
67 passed
```

## Test Philosophy

The tests prefer analytical or known synthetic quantities over smoke tests.
Smoke tests are used only where the behavior depends on GUI, MPI, or optional platform state.

Examples include:

- known binary cubes for volume, surface, and porosity checks
- known grayscale masks for segmentation checks
- known cylinders for fiber diameter checks
- known layered volumes for directional porosity checks
- known output counts for sampling and workflow checks.

## Test Fixtures

Fixtures are generated by the fixture script.

```bash
python scripts/make_test_fixtures.py
```

Pytest also generates fixtures automatically in a temporary directory during normal test runs.

## MPI Note

The MPI contract test launches `mpirun`.
In sandboxed environments, `mpirun` may need permission to open local communication sockets.
